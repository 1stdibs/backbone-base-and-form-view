Backbone.CollectionFormView

new CollectionFormView ()

Similar to Backbone.FormView, except that this form view will iterate over a collection and will produce a row for each model in the collection. Each field in your schema will get a subView for each model in the collection.

Properties:
Name Type Argument Description
collection Backbone.Collection

A Backbone.Collection used to create rows
options object <optional>

Options that can either be set as properties on directly on the view or passed in an object to the constructor.

Properties
Name Type Argument Description
template function | string <optional>

The underscore template to use that wraps the rows. Should contain an empty element that matches the selector for 'rowWrapper' property. By default, this would mean just adding a 'data-rows' attribute to that element.
rowOptions object <optional>

Options to pass to each row when they are initialized
rowSchema object <optional>

The form schema to use for each row
rowConfig object <optional>

Instead of using the rowOptions and rowSchema option, you can define a rowConfig object which works like a BaseView SubViewManager config. Here you can define a constructor as well to use for the row views, in addition to the schema and options.
templateVars object <optional>

Object to use for template variables for the template that wraps each of the collection form rows. To pass a template that is used for each row, use rowOptions.templateSrc
setupOnInit boolean <optional>

True will initialize the row views for each model in the collection. False means the view will wait until render or setupRows is called.

triggerBubble (event, arg) → {Backbone.BaseView}

Like Backbone.trigger, except that this will not only use trigger on the instance this method is invoked on, but also the instance's 'parentView', and the parentView's parentView, and so on. Additional arguments beyond the event name will be passed as arguments to the event handler functions. The view that originated the bubbling event will automatically be tacked on to the end of the arguments, so you can access it if needed.

triggerDescend (event, arg) → {Backbone.BaseView}

Triggers an event that will trigger on each of the instances subViews, and then if a subView has subViews, will trigger the event on that subViews' subViews, and so on. If a subView calls stopEvent and passes the event name, then the event will not trigger on that subViews' subViews. Arguments work in the same manner as they do with triggerBubble.
